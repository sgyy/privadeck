## 简介
本文件面向运维与开发团队，提供媒体工具箱在 Cloudflare Pages 上的静态部署与维护全栈指南。内容覆盖构建配置、PWA（Service Worker、Manifest、离线能力）、CI/CD 自动化部署策略、性能监控与错误追踪、维护与升级流程、故障排除与应急响应，以及日志与分析工具的使用建议。

## 项目结构
媒体工具箱采用 Next.js 16 应用，通过国际化插件增强多语言支持，并以静态导出模式构建为纯静态站点，适配 Cloudflare Pages 的无服务器静态托管特性。关键目录与文件如下：

- 分析与追踪：src/lib/analytics.ts
- WebAssembly 执行：src/lib/ffmpeg.ts

## 详细组件分析

### 静态构建与导出配置
- 关键点
  - output: export：生成静态页面，便于 Cloudflare Pages 直接托管。
  - images.unoptimized: true：禁用自动图像优化，避免运行时处理。
  - trailingSlash: true：为每个路由添加尾随斜杠，利于 CDN 缓存与重定向。
  - 国际化插件：通过 createNextIntlPlugin 包装 next.config，确保多语言路由与静态生成兼容。
  - 跨域隔离头：通过 headers 配置设置 Cross-Origin-Opener-Policy: same-origin 和 Cross-Origin-Embedder-Policy: require-corp，支持 FFmpeg.wasm 的 SharedArrayBuffer 功能。
- 影响范围
  - CI/CD：构建脚本需执行 next build，产物用于 Pages 部署。
  - PWA：静态资源命名与缓存键一致，便于 sw.js 缓存命中。
  - WebAssembly：跨域隔离头确保 FFmpeg.wasm 能够使用 SharedArrayBuffer 进行高效处理。

### PWA 配置与优化
- Manifest（应用元数据）
  - 字段：name、short_name、description、start_url、display、theme_color、background_color、icons、categories。
  - 作用：定义安装到主屏后的应用外观与启动行为。
- Service Worker（缓存与离线）
  - 缓存策略
    - FFmpeg 永久缓存：针对固定版本的 core/wasm 文件，首次加载后持久缓存。
    - HTML 网络优先：保持页面最新，失败时回退缓存。
    - 静态资源缓存优先：JS/CSS/字体/图片等，提升二次访问速度。
  - 清理策略：激活时清理旧缓存键，释放空间。
  - 跨域与非 GET 请求：跳过缓存，避免跨域风险与副作用。
- 离线能力
  - HTML 失败回退：网络异常时返回缓存页面。
  - 静态资源回退：缓存命中失败时返回空响应，避免破坏体验。


### WebAssembly 执行与安全配置
- FFmpeg.wasm 加载机制
  - 单例模式：src/lib/ffmpeg.ts 提供 getFFmpeg() 函数，确保全局只有一个 FFmpeg 实例。
  - ESM 核心：强制使用 ESM 核心版本，因为 @ffmpeg/ffmpeg 在 Web Worker 中创建时指定 type: "module"。
  - Blob URL：使用 @ffmpeg/util 的 toBlobURL 将核心文件转换为 blob: URL，满足 CSP script-src 要求。
  - 内存优化：通过 WORKERFS 挂载输入文件，避免内存拷贝，提升处理效率。
- 跨域隔离支持
  - COOP/COEP 头：next.config.ts 中配置 Cross-Origin-Opener-Policy: same-origin 和 Cross-Origin-Embedder-Policy: require-corp。
  - SharedArrayBuffer：启用后可使用 SharedArrayBuffer，显著提升 FFmpeg.wasm 的性能。
  - 兼容性：在不支持 SharedArrayBuffer 的环境中提供降级提示。
- 安全配置
  - CSP 兼容：blob: URL 支持允许 Web Worker 动态导入核心文件。
  - 资源限制：通过缓存策略限制 FFmpeg 核心文件的下载频率。
  - 错误处理：完善的异常捕获与资源清理机制。

## 性能考虑
- 构建期
  - 静态导出减少运行时渲染压力；禁用图片优化降低构建复杂度。
  - 尾随斜杠有助于 CDN 缓存命中与规范化链接。
- 运行期
  - Service Worker 缓存策略按资源类型分流，显著提升重复访问性能。
  - FFmpeg 资源永久缓存，避免频繁下载大体积 WASM。
  - 跨域隔离支持 SharedArrayBuffer，提升 WebAssembly 执行性能。
- 监控与优化
  - 结合 Pages 边缘日志与 gtag 数据，识别慢页面与高错误率路径，针对性优化资源或算法。

## 故障排除指南
- Service Worker 未生效
  - 检查 sw.js 是否随静态资源一起发布；确认浏览器已注册并处于激活状态。
  - 若更新缓存键或策略，请清除旧缓存或等待激活清理。
- FFmpeg 加载失败
  - 确认网络可达性与跨域策略；检查补丁是否正确应用。
  - 验证跨域隔离头配置是否正确设置。
- 分析事件未上报
  - 确认 gtag 注入与可用性；检查隐私裁剪是否导致关键字段缺失。
- 部署失败
  - 确认构建命令与产物路径；核对 Pages 设置中的构建命令与输出目录。
- WebAssembly 性能问题
  - 检查浏览器是否支持 SharedArrayBuffer。
  - 验证 COOP/COEP 头是否正确配置。
  - 确认 FFmpeg 核心文件缓存是否正常工作。

### 安全配置最佳实践
- Content-Security-Policy
  - 确保 script-src 包含 blob: 以支持 Web Worker 动态导入。
  - 连接策略限制到必要的第三方服务。
- 跨域隔离
  - 正确配置 Cross-Origin-Opener-Policy: same-origin。
  - 设置 Cross-Origin-Embedder-Policy: require-corp。
  - 验证 HTTPS 环境下的安全策略。
- WebAssembly 安全
  - 限制 WebAssembly 内存使用。
  - 监控异常的内存增长。
  - 实施适当的超时机制防止长时间运行的处理任务。
